# TUSUR
Модуль 12 и возможно квалификационная работа

Вариант 2. LLM-сервис с RAG
Задача: реализовать backend-сервис на базе LLM с RAG-пайплайном для ответов на вопросы по заданному набору документов.

Задание 1. Корпус документов и постановка задачи

1.1. Определите предметную область, например:

внутренние регламенты компании;
документация по продукту;
учебные материалы.
1.2. Подготовьте небольшой корпус текстов (PDF / Markdown / HTML / обычный текст) — 5–20 документов.

1.3. Сформулируйте типовые пользовательские запросы:

информационные вопросы;
уточняющие вопросы по конкретным пунктам документов.
Задание 2. RAG-пайплайн

2.1. Реализуйте следующие шаги:

разбиение документов на фрагменты (чанки) с учётом размера контекста;
построение эмбеддингов для фрагментов (например, через sentence-transformers или иные библиотеки, согласованные с преподавателем);
хранение эмбеддингов во внутреннем векторном хранилище (in-memory, FAISS, Chroma и т.п.).
2.2. Реализуйте функцию поиска релевантных фрагментов по запросу пользователя (top-k).

2.3. Реализуйте функцию формирования промпта для LLM:

включение вопроса пользователя;
добавление найденных фрагментов;
системное сообщение с инструкциями (отвечать строго на основе контекста, указывать, если информации не хватает).
2.4. Подключите LLM:

локальную/открытую модель или API;
реализуйте интерфейс вызова модели.
Задание 3. Backend-сервис

3.1. Реализуйте REST-сервис на FastAPI (или аналогичном фреймворке) с эндпоинтами:

GET /health — статус сервиса;
POST /ask — вход: JSON вида:
{
"question": "Как оформить возврат товара?"
}
Выход:
{
"answer": "Текст ответа модели...",
"sources": [
{
"doc_id": "policy_01",
"score": 0.87,
"snippet": "Фрагмент документа..."
}
]
}
3.2. Обеспечьте:

обработку ошибок LLM / векторного хранилища;
ограничение длины вопроса;
логирование запросов минимум в консоль.
Задание 4. Подготовка и запуск сервиса

4.1. Подготовьте проект к запуску:

установку всех библиотек (FastAPI, LLM / эмбеддинги, FAISS / Chroma и т.п.);
загрузку документов и построение индекса;
подключение конфигурации через переменные окружения при необходимости.
4.2. Реализуйте запуск сервиса локально, например:

pip install -r requirements.txt
uvicorn main:app --reload

4.3. При необходимости предусмотрите:

хранение документов и индекса в отдельных каталогах проекта;
передачу секретов и ключей через переменные окружения без их сохранения в коде.
Задание 5. Тестирование и документация

5.1. Подготовьте набор тестовых запросов и продемонстрируйте:

ответы модели, опирающиеся на предоставленные документы;
поведение модели, если информации не хватает (например: «Информации в документах недостаточно...»).
5.2. Оформите README.md:

описание архитектуры RAG-пайплайна;
формат запросов/ответов;
инструкции по установке зависимостей и запуску сервиса;
ограничения и возможные улучшения.
